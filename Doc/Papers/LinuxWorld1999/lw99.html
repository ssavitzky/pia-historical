Goals:

 

The goals of the PIA project are:

  1. to make it easy to develop document-processing applications
  2. by providing a single language for content and processing
  3. to give authors and users more power over their documents
We want to start building a community
of users, information designers, and programmers
working together on a shared, standards-based, open platform.
Documents and the Web

What do people do with documents?

  • Fill out and process forms
    • checks, purchase requests, approvals
    • Sometimes ``processing'' = signing.
  • Create and sign legal documents
  • Edit and publish manuscripts
  • Memos: send, recieve, copy, file, ...
  • Letters, bills, junk mail, magazines, postit notes, ...

Processing:

  • signing: approving, authorizing, ...
  • storing: indexing, filing, stacking, boxing, ...
  • retrieving: searching, skimming, ...
  • excerpting, annotating, summarizing, ...
  • presenting, performing, parodying, ...
  • deleting, shredding, ...

Documents: From Paper to the Web

 

  • Before computers, documents were ink on paper
    • All these things were easy
  • Computers changed things: documents became data
    • Doing anything requires an application.
    • sharing requires everyone to have the same application
  • The Web changed things again.
    • HTML is a shared language

  • But processing still requires applications.

The Web is Open

Why did the Web grow so explosively?
Because it is open source
  • Both web pages (HTML) and servers
Because our language extends HTML and XML,
it ought to be open, too.
Besides,
Open Source is best, fastest way to evolve the most reliable system

Time to make a release

We think we have a good design.
We have working code.
So we've reached the point where Eric Raymond says we should release it as Open Source.

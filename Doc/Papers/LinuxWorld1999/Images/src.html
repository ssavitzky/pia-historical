/PIA/src/

.../pia/src

[..] [files]

This is the top-level directory for source code (other than PIA Agents) generated by members of the PIA Project at CRC. This includes, for example, the Java source code for the PIA itself, tools and other applications, and so on.

This directory subtree is not included in the PIA binary distribution.

Annotated Contents

app/
Separate, stand-alone applications meant to be executed from a command line or other shell. Utilities.
cgi/
CGI scripts and programs, meant to be installed in .../cgi-bin on some server.
java/
Java packages. (Note that Java does not distinguish between a class library and an application, so we can put them all in one place.)
perl/
PERL packages. (The PERL implementation is obsolete, but will probably still run. You may find it interesting for historical reasons. Eventually it would be useful to bring it back up to date.)

Typical Make Targets

make all
Make all binaries.
make doc
Make documentation (mainly installed in ../Doc and its subdirectories).
make install
Install executables in pia/bin, and libraries in pia/lib.

Naming Conventions

In general, source code sub-directories that correspond to ``packages'' or ``modules in object-oriented programming languages like Perl and Java obey the naming conventions of those languages. So, for example, the subdirectories of perl have uppercase module names, while the subdirectories of java have lowercase package names.

PIA Short-Course Syllabus

This is a syllabus for a one-week course on the PIA, with an emphasis on porting to an embedded system.
Note:
Cygnus is releasing an Open Source RTOS: http://www.cygnus.com/ecos/. In a year or two, it might be a viable alternative to VxWorks.

Syllabus

Monday 10:00 am: Agenda and Introductions

Monday am: Introduction to the PIA Framework

  1. General Introduction
    • Clients, servers, and proxies (terminology).
  2. What Is the PIA?
    • (Proxy) Server
    • Agents
    • InterForms
  3. Sample Agents
  4. Structure of an Agent
    • Agent object
    • Agent Names and Types
    • InterForm and Data directories
    • How findInterForm works
    • Extension and directory search paths

Monday pm: Building Your Own Agents

  1. Agent Files
    • initialize
    • home and its inclusions
  2. Overview of the XHTL language
  3. SGML Syntax
    • Identifiers and Namespaces
    • Text
    • Elements and Attributes
    • Entities
    • Processing Instructions
    • Declarations and Comments
  4. Tagsets and DTD's
    • What's a DTD?
    • What's a Tagset?
    • How do you read a Tagset

Tuesday am: Object-Oriented Design and Java

  1. Object-Oriented Design
  2. Frameworks
  3. Java Language
    • Key features: interfaces, classes, libraries.
    • How it differs from C
    • How it differs from C++
    • Where to find out more

Tuesday am: Programming Custom Agents

  1. Writing custom InterForms.
  2. Extending GenericAgent to add functionality.
    • do this when you need to modify the URL, or to obtain content from an unusual source (e.g. a device).
    • e.g. DOFS, Logo
  3. Extending GenericHandler to add functionality.
    • do this when you need to perform an unusual operation on a piece of an active document, or to obtain HTML content from an unusual source.
  4. Implementing new Agent, Handler, etc. classes.

Wednesday am: PIA Server internals

  1. Package Overview
  2. PIA major classes and interfaces
    • Agent, GenericAgent
    • Acceptor, Resolver
    • Transaction, HTTPRequest, HTTPResponse
    • Content
    • sub-packages
  3. How the Resolver Works
    • Transaction
    • Features and Criteria
    • actOn methods
    • handle methods

Wednesday pm: internals (continued)

  1. Agency Agent.
    • handles defaults
    • performs server management functions.
  2. DPS major classes and interfaces
    • DOM interfaces (and what's going to change)
    • DPS interfaces
    • sub-packages

Thursday am: Porting to C

  1. Possible Porting Techniques
    • JVM on target platform
    • C port
  2. What is essential?
  3. What can be left out?
  4. Translating Java to C
    • Translating method calls
    • Translating an Interface
      (to a struct of pointers to functions).
    • Translating a Class
      (to a struct plus interface functions).
    • Memory Management.
  5. Available Resources.
    • Mozilla -- they probably have a DOM in C!

Thursday pm: VxWorks Issues / Network services

  1. Operating Systems
    • VxWorks
    • RT-Linux
    • eCos
  2. PhotoNet and other Network Services

Friday am: Camera Communitation Possibilities

  • Michael Baxter

Friday pm: Discuss Implementation/Porting Plan

  • open discussion

Installing the PIA

This manual describes how to install the Platform for Information Applications (PIA). This information is directed at PIA users. Developers intending to create applications should also consult Authoring Agents.

Topics covered include:

  • Before You Begin
  • Obtaining the Files
  • Installing the Files
  • Running the PIA
  • Questions, Problems and Comments

For information on use of the PIA, see the PIA Document List.

Before You Begin

You need the following software to run the PIA:

  • Java JRE 1.1 or higher is needed to create and run applications.
    JDK is needed to modify the PIA code. JDK can be obtained from http://www.javasoft.com
  • An operating system that supports sockets and long filenames.
    We have tested the PIA on Linux (Red Hat 4.2), Windows 9X, and Solaris on SPARC.
  • The tar and gunzip utilities.
    These are required for unpacking the files.

Obtaining the Files

The PIA files are available via HTTP, from http://www.risource.org/PIA/.

The latest stable releases are maintained in compressed form in the directories ftp://www.risource.org/PIA/Pia_bin.tgz and ftp://www.risource.org/PIA/Pia_src.tgz.

Installing the Files

The files are supplied in the form of a gzip'ed tar file. Set your working directory to the intended the parent directory for the PIA files. Extract the files using tar.

tar xvf pia_bin_2_0b

This creates the directory pia/ under your home directory.

Setting Up

The PIA uses two environment variables:

  • PIA_DIR indicates the location of the PIA directory just installed.
  • USR_DIR indicates the location to which the PIA writes its data files.

On Unix, set the PIA_DIR to $HOME/pia or whatever installation location you chose above.On Windows, set the PIA_DIR to drivename:\pia.

USR_DIR is automatically created by the PIA. You are free to reset its location if desired.On Unix, the default directory for USR_DIR is ~/.pia. On Windows, the default directory is c:\PIAUSERS\username.

If you use the -u command line option to specify USR_DIR, that setting takes precedence over the one you specify in the environment variable.

When the PIA starts, it looks for the pia.props under ~/.pia/Config. You can create a customizable copy in your home directory under ~/.pia/Config by copying pia/Config/pia.props and modifying it using a text editor. Note that this is USR_DIR.

The default settings of the PIA are contained in pia/Config/pia.props.

If you start the PIA with a -u userdir option, the PIA looks for the pia.props in userdir/Config.

It is convenient to put the directory pia/bin into your shell's search path.

Running the PIA

On UNIX, once you have set up the PIA_DIR and USR_DIR environment variables, you can launch the PIA with the command: PIA_DIR/bin/pia.This is a one-line shell script that runs pia_wrapper. A complete list of command-line options can be found by running PIA_DIR/bin/pia -? at the shell prompt.

Under Windows 95 and NT, launch the PIA with the command: PIA_DIR/bin/pia.bat.

This is a batch file that makes use of the PIA_DIR and USR_DIR environment variables. It is equivalent to the following:java -classpath CPATH crc.pia.Pia -root PIA_DIR -u PIA_HOME

Note the -root and -u options. The -p option can be used to set the port.

Using the PIA

The PIA is essentially a Web server. Assuming you are running on a machine called PiaHOST using piaPORT, simply browse to http://piaHOST:piaPORT/

Some of the functions work best if you set your browser's proxies to the PIA as well. In most cases, http://localhost:8888 will work.

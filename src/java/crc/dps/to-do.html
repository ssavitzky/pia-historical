  1. Bug: Parsing: <tr> gets lost
    <tr> ends the previous <th>, but neither the expected </tr> nor the <tr> show up. An entity before an inserted </td> also causes havoc -- it turns into a comment! (probably bad node type). (1998-6-26)
  2. Bug: slides: <h2> doesn't work
    It produces no output, but the entity doesn't seem to get defined. (1998-6-26)
  3. Bug: replaceChild when redefining entity or handler
    BasicTagset and BasicEntityTable have to do a replaceChild when replacing a binding. (1998-6-26)
  4. Hole: Must define tags at parse time to cache trees
    In order to parse correctly, you have to have the whole DTD in advance. (1998-6-25)
  5. Impr: Upgrade handle methods.
    The static handle(element) methods in handler subclasses should pick up a static instance if possible, rather than just calling new. (See also ''Impr'' of 1998-06-15.) (1998-6-25)
  6. Goal: (8-1) Run Most Agents
    Be able to run all of the agents in daily use (i.e. history, etc., but not the wacko stuff in on the form server). (1998-6-24)
  7. Goal: (7-1) Able to process slides
    Be able to process .../Reviews/slides-199807.html in time to show it at the review. Requires <actor> (1998-6-24)
  8. Bug: Flag unimplemented legacy options.
    Unimplemented legacy options in <get> etc. have to be brought to the user's attention with error messages. Requires a mode flag or, better, dispatching via the legacy handler first. (1998-6-24)
  9. Impr: Character Entities should be defined in tagset
    Should have defined value; copy value into references to short-circuit lookup. TextUtil.expandCharacterEntities can then make use of this. (1998-6-22)
  10. Hole: ToParseTree should take a tagset
    Tagset is needed to attach actions to constructed nodes. (1998-6-22)
  11. Impr: Tagset should extend EntityTable.
    This will make it easier to define entities and tags in the same DTD. (1998-6-19)
  12. Hole: Character entity class, handler
    Having a separate node class and handler class for character entities would simplify inserting and expanding them. Would break some legacy code, though. (1998-06-19)
  13. Impr: PUT_VALUE action code needed
    Get a value for the node and put that. Needs a corresponding putValue(node, context, output) in Action; the default should be to copy the node. (1998-06-18)
  14. Hole: ParseTreeDocument & XML nodes.
    ...are still unimplemented. (1998-06-18)
  15. Hole: PIA transition from old to new interp.
    Agents need to know which interpretor (and tagset, of course) they should be using. Hooks can be distinguished by type. (1998-06-18)
  16. Hole: PIA Content in push mode.
    crc.pia.Content currently operates in pull mode, expecting to read from an input stream. For the DPS it needs to run in push mode, giving the DPS a suitable output stream and letting it run. (1998-06-18)
  17. Hole: PIA namespaces and data types
    During the transition Agent, Transaction, etc. need to be able to return both SGML and DOM data as needed. (1998-06-18)
  18. Prop: New InterForm extensions (?).
    .mh
    ``Meta-HTML'' -- resulting in a MIME type of text/html
    .mx
    ``Meta-XML'' -- resulting in a MIME type appropriate for XML.
    .md
    ``Meta-Data'' -- the MIME type is computed by the file
    .mt
    ``Meta-Transaction'' -- the entire headers are computed.
    (1998-06-18)
  19. Impr: ISO 8601 date representation (dashes)
    See this summary. The main things needing fixing are the entity table initialization, and dates as presented by the History agent's hook. (1998-06-18)
    -- Should add timezone and UTC time/date as well.
  20. Impr: Entities need to be initialized by the Tagset.
    The justification for this is that strictly speaking entities are defined in the DTD. (1998-06-18)
  21. Impr: Caching for subProcess stack
    ContextStack should have a free link to a free sub-processor, to be used instead of allocating one. (1998-06-18)
    -- subProcess clears free; caller does p.cleanup() when finished to restore it.
  22. Hole: NamedNodeList wrapping a Table
    Want to be able to wrap a Table as a NamedNodeList. This will provide the DPS equivalent of crc.sgml.AttrWrap. (1998-06-16)
  23. Impr: tag handler classes need loader methods
    Tagset shouldn't have to know how to load classes for handlers. Having static methods for this would allow tag handlers to manage their own name spaces. (1998-06-16)
  24. Check: copying
    Non-processed parse trees (e.g. <else>) should never be copied. Copying should use putNode if the node actually has children; recursion if the children are unparsed. (1998-06-15)
  25. Impr: Filter parse tree input
    Filter should be able to read a serialized parse tree. (1998-06-15)
    -- It should do so if the input file ends in ``.obj''.
  26. Query: <test zero> non-number</> = ?
    Not clear whether non-numerics should test as zero or not. The most consistant thing would be for whitespace to return true, non-whitespace to return false. (1998-06-15)
  27. Hole: Legacy Handlers

  28. Hole: New Handlers

    done: subst if else then elsf test
  29. Check: attribute and tag case sensitivity
    Nothing is careful about cannonical attribute and tag names yet.
  30. Impr: NamedActiveNode values
    Named nodes should have defined, assigned and default values. Only defined value should be the children; it would be used for the definition of a name. Want to be able to share assigned values; the NodeList of an assigned value would be a copy (just like we presently do for attribute lists, and for the same reason).
  31. Hole: BasicParser: content parse control
    Currently Parser does not check syntax of current node to see whether to look for entities or elements in the content.
  32. Hole: SGMLParser: content models
    Currently Parsers and Tagsets do not have any notion of content model.
  33. Impr: ParseTreeElement hasActiveAttributes
    ParseTreeElement should keep information about whether any of its attributes contain entities that need to be expanded. Should be much faster, since most don't and it's known at parse time.

    The other possibility is lazy evaluation -- don't even look at them until they're needed when converting an attribute's value to a string.


  34. Wish: toNextNode
    Might want flag in Input's that tells whether the current node has been processed yet. (really relevant only for toNextNode, to keep track of whether we've enumerated the children yet)
  35. Wish: ParseTreeIterator
    We will want a ParseTreeIterator that implements both Input and Output (and of course TreeIterator). The main use would be constructing an node and then processing it.
  36. Wish: ParseTreeGenericNode
    Eventually there should be a ParseTreeGenericNode, basically Token, so that we can build trees with arbitrary node types.
(End of list)

Description of Keywords

(Listed in order of decreasing urgency.)
Bug
A (typically small) problem with a simple, unambiguous description of its symptoms.
Hole
A piece of missing, but important, functionality.
Goal
A target capability (includes a date)
Check
Something to check for; a potential stumbling block or place where the implementor may have cut corners in the heat of the moment. For example, Y2K problems.
Improvement
A desirable improvement.
Proposal
A proposal that will need to be discussed further.
Wish
A desired improvement of low priority.

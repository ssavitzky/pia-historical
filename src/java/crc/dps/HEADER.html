InterForm Interpretor and Actors

[..] [files] ( about, to-do, done )

This directory contains the all-new, all-singing, all-dancing, SGML-based, buzzword-compliant Document Processing System.

Also see the old InterForm Interpretor.

Annotated Contents

Packages

active/
Parse trees made out of ``active nodes''.
aux/
Auxiliary classes, not meant for use by the casual programmer.
handle/
The Handle subclasses that implement actions (operations) attached to actors. It turns out that tags is a bad name for a subdirectory, since it's the name used for the ``tag file'' created by the ctags utility. CVS ignores it, and various cleanup and maintenance operations are likely to flag it as deletable.
input/
Sources of nodes.
output/
Destinations for nodes.
parse/
Parsers (i.e. Readers for nodes).
tagset/
Tagsets (lookup tables for syntax and semantic Handlers).
test/
test cases

Interfaces

Contexts

Context
An environment in which processing is done. Contains an Input, an Output, and an EntityTable.
Processor
A document processor. Both a consumer and producer of Nodes
TopContext
A top-level context. (An implementation of TopContext is normally a Processor as well.)

Handlers

Action
Semantic handlers for active nodes.
Active
Interface for ``active'' objects (i.e. objects with an Action).
Syntax
Interface for node syntax handlers.
Handler
Combination of Syntax and Action handlers.

Bindings

EntityTable
A lookup table for Entity values. Supports context chaining.
Tagset
A lookup table for Handlers. Supports context chaining.

Traversal

Cursor
An interface that allows anything with a ``current node'' to be handled efficiently. The parent for Input and Output.
Input
A source of nodes, essentially a tree traverser.
Parser
An Input that parses nodes from a character stream.
Output
A tree constructor.

Classes

``Abstract'' in a name marks an abstract base class; ``Basic'' marks a class with full but perhaps limited functionality.

DocumentProcessor
The basic implementation of TopContext.
BasicProcessor
The basic implementation of Processor.
Filter
A main-program wrapper that allows a DocumentProcessor to be used as a filter for (gasp!) processing documents.
